What is the MultiFungible SDK and API ?

The MultiFungible software development kit (SDK) and application-specific interface (API) is a cross-platform library written in C++ that allows to create and manage non-fungible tokens (NFTs) and semi-fungible tokens (SFTs) in the Multivers X blockchain.

Example:
Wallet1: erd1catxddevywldkrwujn4aj6e4d9ruhrrwtw8yfe3pdk9p9awmlnzqpsc5lp
Wallet2: erd1ahxq4s4dtsp92tdxy2g5mutzlzrnz59seavmrt27kqpujl6v37psnwuat2
Issued collection: ABCD-012345
Issued token: ABCD-012345-01
Creation stopped successfully
Sent 10 units of ABCD-012345-01 to erd1ahxq4s4dtsp92tdxy2g5mutzlzrnz59seavmrt27kqpujl6v37psnwuat2 successfully

Installation and usage

This library can be installed as an SDK on Windows and Linux. It can also be compiled as a dynamic-linked library (DLL) or a linux dynamic library (.so).

Windows installation

If you wish to compile the library yourself, you should install the following libraries

OpenSSL
  • You need to install and compile OpenSSL 3.0.8 or higher to run Multifungible.
  • Follow the tutorial to install Openssl on windows. Here is a small summary of the steps:
  • Open the x64 Native tools command prompt
  • Run: perl Configure VC-WIN64A --prefix=c:\local\openssl3.0.8
  • REMOVE ALL OCCURENCES OF -DOPENSSL_USE_APPLINK from the make file (so we don't need to use applink.c in our applications).
  • Run: nmake
  • Run: nmake install

Sodium
  • You need the libsodium library binaries to run Multifungible.
  • Download here the latest pre-builts for mingw compiler.
  • Uncompress the .tar.gz into your location of choice.

Linux installation

  • To compile the library on Linux along with your project, you will need the latest version of openssl 3 and libsodium.
  • To install openssl headers and library, and the libsodium headers, run the install.sh script.
  • To install the libsodium library, go to this page, and download the latest stable release. Then, run the commands ./configure, make && make check and sudo make install

Compilation

Now that you have the necessary libraries installed, you are going to run cmake to compile the Multifungible library. A compilation script is provided for windows and linux. Please, modify it and set the value of the following macros to successfully compile the product:

  • CMAKE_C_COMPILER: Select the location of the C compiler. Example: -DCMAKE_C_COMPILER="C:/TDM-GCC-64/bin/gcc.exe" on windows. usually, no need to set this on linux.
  • CMAKE_CXX_COMPILER: Select the location of the C compiler. Example: -DCMAKE_CXX_COMPILER="C:/TDM-GCC-64/bin/g++.exe" on windows. usually, no need to set this on linux.
  • MULTIFUNGIBLE_NETWORK: Select the network in which the library will run (Local, Testnet, Devnet, Mainnet). Example: -DMULTIFUNGIBLE_NETWORK=Mainnet (without ""). Compulsory.
  • MULTIFUNGIBLE_CONFIG_FILE: Select the location of the config file that the library will use during execution. Example: -DMULTIFUNGIBLE_CONFIG_FILE=./config.toml (without ""). Compulsory.
  • LIBSODIUM_INCLUDE_PATH: Select the path to the headers of the libsodium library. Example: -DLIBSODIUM_INCLUDE_PATH="C:/libsodium/include". No need to set this on linux.
  • LIBSODIUM_LIB_PATH: Select the path to the compiled libsodium library (libsodium.a). Example: -DLIBSODIUM_LIB_PATH="C:/libsidum/lib". No need to set this on linux.
  • OPENSSL_INCLUDE_PATH: Select the path to the headers of the openssl library. Example: -DOPENSSL_INCLUDE_PATH="C:/openssl3.0.8/include". No need to set this on linux.
  • OPENSSL_LIB_PATH: Select the path to the compiled openssl library (libcrypto.lib and libssl.lib)/ Example: -DOPENSSL_LIB_PATH="C:/openssl3.0.8/lib". No need to set this on linux.

Issue SFT Collection

returnCodeAndChar issueSFTCollection

Issues an SFT collection on the blockchain with the attributes provided in the arguments.

Arguments:
Name Type Default Value Description
p_walletName char * Required Name of the PEM file on the hard-drive.
p_password char * Required Password to decrypt the PEM file.
p_sftName char * Required Name to give to the collection. Must be alphanumeric, between 3 and 20 characters.
p_sftTicker char * Required Ticker to give to the collection. Must be alphanumeric, between 3 and 10 characters.
p_canFreeze bool false Allow the collection to be freezed by an account that possesses the right to do so.
p_canWipe bool false Allow a token from the collection to be wiped by an account that possesses the right to do so.
p_canPause bool false Allow to pause the issuance of tokens of the collection by an account that possesses the right to do so.
p_canTransferNFTCreateRole bool false Allow the account that has the ability to issue tokens on this collection to transfer this ability to another account. Only one account can have this ability at a given time, for the collection.
p_canChangeOwner bool false Allow to change the owner of the collection by an account that possesses the right to do so.
p_canUpgrade bool false Allow to upgrade the roles that an account possesses on the collection by an account that possesses the right to do so.
p_canAddSpecialRoles bool false Allow to add roles to the collection by an account that possesses the right to do so.

Return Value:

returnCodeAndChar structure containing the following fields:

Name Type Description
retCode int 0 if the operation was successful, 1 if it wasn't.
message char * Collection ID if the operation was successful. Error information if it wasn't.

Usage example:


Result:

ABCD-012345

p_uri char * URL to a media file representing the token in some way. The media should be of a type supported by Multivers X.

returnCodeAndChar structure containing the following fields:

Name Type Description
retCode int 0 if the operation was successful, 1 if it wasn't.
message char * Token ID if the operation was successful. Error information if it wasn't.
